### 1. 任务配置区的详细交互逻辑

任务配置区是整个应用最核心的区域，设计目标是**“像填闹钟一样简单，同时支持进阶配置”**。

#### 页面布局（从上到下）

**区域 A：Binary（执行程序）**
- 输入框 + 右侧「浏览」按钮
- 支持两种输入方式：
  - 直接输入命令名（如 `codex`、`claude`）
  - 点击「浏览」选择本地可执行文件
- **智能提示**：输入时实时搜索系统 PATH 中的常见命令（Codex、claude、node、python 等）
- 验证逻辑：
  - 输入后自动检测该 binary 是否存在
  - 存在则显示绿色「可用」标签
  - 不存在则显示黄色「未找到」，但仍允许保存（用户可能在其他机器上运行）

**区域 B：参数（Args）**
- 多行文本输入框（类似代码编辑器，但更简洁）
- 支持换行输入多个参数
- **模板变量提示**：在输入框下方显示可用的变量（如 `{{date}}`、`{{datetime}}`），点击可直接插入
- 实时预览：输入框下方显示最终执行的完整命令预览（只读）

**区域 C：环境变量（可折叠）**
- 默认收起，点击「添加环境变量」展开
- 使用 Key-Value 列表形式
- 支持快速添加常用变量（如 `CLIPROXYAPI_API_KEY`）
- 每个变量右侧有删除按钮
- 支持从系统环境变量一键导入（可选功能）

**区域 D：失败重试设置（可折叠，默认展开）**
- 使用 **Segmented Control**（分段控制器）选择重试间隔：
  - 选项：1分钟 / 2分钟 / 5分钟 / 10分钟
- 重试次数固定为 **最多 3 次**（不让用户修改，保持简单）
- 右侧显示说明文字：「失败后会按间隔自动重试，最多重试 3 次」

#### 交互细节

- **实时验证**：切换焦点或输入停止 500ms 后自动验证 binary 是否可用
- **保存前检查**：点击保存时，如果 binary 未找到，弹出温和提示：「这个命令在本机可能不存在，确定要保存吗？」
- **常用模板**：右上角提供「常用任务模板」下拉菜单（例如「Codex 新闻报告」「Claude 简单问候」），点击后自动填充 binary 和参数
- **参数高亮**：如果参数中包含敏感信息（如 API Key），输入时默认隐藏（可切换显示）

### 2. 日志页面的具体展示形式

日志页面采用**时间轴 + 卡片**的混合形式，既美观又实用。

#### 页面结构

**顶部工具栏**：
- 搜索框（支持按任务名称、输出内容搜索）
- 筛选器：全部 / 成功 / 失败 / 重试中
- 时间范围选择（今天 / 最近 7 天 / 自定义）

**日志列表**（时间倒序）：

每条日志使用**横向卡片**展示，包含以下信息：

| 元素           | 内容                              | 样式建议                  |
|----------------|-----------------------------------|---------------------------|
| 时间           | 执行开始时间                      | 较小字体，灰色            |
| 状态           | 成功 / 失败 / 重试中              | 彩色徽章（绿/红/橙）      |
| 任务名称       | 闹钟名称                          | 粗体                      |
| 执行命令       | binary + 简短参数预览             | 等宽字体，截断显示        |
| 耗时           | 执行时长                          | 小字                      |
| 重试次数       | 本次执行共重试了几次              | 小字（仅失败时显示）      |

**交互行为**：
- 点击任意日志卡片 → **右侧或下方展开详情面板**
- 详情面板内容：
  - 完整执行命令
  - 环境变量（脱敏显示）
  - 标准输出（stdout）
  - 错误输出（stderr）
  - 开始时间、结束时间、总耗时
  - 重试记录（如果有多次重试）

**视觉设计**：
- 成功日志使用柔和绿色边框
- 失败日志使用柔和红色边框
- 重试中的日志带有轻微动画（呼吸灯效果）
- 空状态时显示可爱插图 + 文案：「今天还没有执行记录哦～」

### 3. 托盘图标 + 右键菜单设计

由于是后台运行工具，**系统托盘**体验非常重要。

#### 托盘图标设计

- **图标风格**：使用简洁可爱的闹钟图标（带小动物元素或岛屿风格）
- **状态区分**：
  - 正常运行：彩色图标
  - 有任务正在执行：图标轻微旋转或带小 loading 效果
  - 存在失败任务：图标右上角显示小红点
- 支持 **Dark / Light** 模式自动适配

#### 右键菜单（Context Menu）

点击托盘图标右键后弹出菜单，推荐以下结构：

```markdown
callai
├── 显示主窗口          （点击后打开/激活主界面）
├── 立即执行全部闹钟     （危险操作，需二次确认）
├── ─────────────────
├── 新建闹钟
├── 查看日志
├── ─────────────────
├── 暂停所有闹钟
├── 恢复所有闹钟
├── ─────────────────
├── 设置
├── 关于 callai
├── 退出
```

**菜单交互细节**：

- **显示主窗口**：最常用操作，放在第一位
- **立即执行全部闹钟**：仅在调试或紧急情况下使用，点击后弹出可爱确认弹窗
- **暂停所有闹钟 / 恢复所有闹钟**：一键控制所有任务的运行状态
- **退出**：退出应用（会停止所有后台任务）

**左键点击行为**：
- 默认行为：点击托盘图标 → 显示/隐藏主窗口（类似很多工具类应用）

### 4. 补充：其他重要交互与体验细节

**全局交互原则**：
- 所有危险操作（立即执行、删除闹钟、退出应用）都需要二次确认，且使用温和的文案。
- 长时间运行的任务在执行时，首页对应卡片会显示「正在执行…」状态。
- 应用启动时，如果有闹钟即将触发，会在托盘显示通知（可选，可在设置中关闭）。

**空状态设计**：
- 首次打开应用时，首页显示可爱空状态：
  - 插图：一个小动物在等闹钟响
  - 文案：「还没有设置任何闹钟哦～ 点击右下角按钮创建一个吧！」

**通知（可选增强）**：
- 任务执行失败时，可选择是否弹出系统通知（默认关闭，避免打扰）
