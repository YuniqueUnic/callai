**✅ Callai 使用场景、操作逻辑、步骤与 UI 设计说明**

以下内容按**产品实际使用流程**组织，便于后续开发和设计参考。

---

### 1. Use Cases（使用场景）

| Use Case | 用户角色 | 场景描述 | 频率 | 优先级 |
|---------|----------|----------|------|--------|
| **日常额度窗口维护** | 重度 AI 用户 | 每天固定时间自动执行轻量任务，保持 Claude / Codex / ChatGPT 的滚动窗口新鲜 | 每天 | ★★★★★ |
| **手动立即执行任务** | 所有用户 | 想立刻测试某个任务是否能正常执行 | 偶尔 | ★★★★ |
| **新建 / 编辑闹钟** | 所有用户 | 创建一个新的定时任务，或修改已有闹钟的时间、命令、参数 | 中等 | ★★★★★ |
| **查看执行历史与日志** | 进阶用户 | 查看某个闹钟最近的执行情况、是否成功、重试记录 | 偶尔 | ★★★ |
| **暂停 / 启用闹钟** | 所有用户 | 临时不想让某个闹钟运行（例如假期） | 偶尔 | ★★★★ |
| **快速复制闹钟** | 进阶用户 | 想基于现有闹钟快速创建一个相似配置 | 低 | ★★ |

**核心 Use Case** 是第 1 个：**“给 AI 定闹钟”**，其他功能都围绕它服务。

---

### 2. 操作逻辑（核心设计原则）

- **以“闹钟”为核心隐喻**：用户不是在配置“任务调度器”，而是在设置一个“闹钟”。
- **闹钟与任务强绑定**：创建一个闹钟时，必须同时配置要执行的任务（binary + 参数）。
- **极简优先**：主流程控制在 3 步以内完成。
- **后台静默运行**：设置完成后，应用可以最小化到托盘，后台继续执行。
- **可爱但清晰**：使用 animal-island-ui 的温暖风格，但重要操作要有明确反馈。

---

### 3. 主要操作流程与步骤

#### **主流程：新建一个 AI 闹钟（核心流程）**

**步骤：**

1. **打开应用** → 首页展示当前已创建的闹钟列表（卡片形式）
2. **点击右下角「+ 新建闹钟」**（大圆形可爱按钮，类似动森的添加按钮）
3. **进入「新建闹钟」页面**，按以下顺序填写：
   - **闹钟名称**（例如：上午额度刷新）
   - **触发时间**（支持两种方式）：
     - 简单模式：选择「每天」+ 时间点（可多选，如 8:00、13:00、18:00）
     - 高级模式：直接输入 cron 表达式（可折叠）
   - **执行任务**（核心配置区）：
     - Binary：支持下拉选择常用命令（Codex、Claude 等），或手动输入完整路径
     - 参数：文本框，支持多行输入
     - 环境变量（可选展开）：Key-Value 形式添加
   - **失败重试**（默认开启）：
     - 重试次数：最多 3 次（固定）
     - 重试间隔：可选择 1min / 2min / 5min / 10min（默认 2min）
4. **点击「保存」** → 闹钟创建成功，回到首页列表

**创建后行为**：
- 应用后台持续运行
- 到时间自动执行任务
- 执行结果记录到日志

---

#### **其他重要流程**

**流程 A：手动立即执行（测试用）**
1. 在闹钟卡片上点击「立即执行」按钮
2. 弹出确认框（可爱风格）
3. 执行完成后，显示执行结果（成功 / 失败 + 耗时）
4. 可一键跳转查看详细日志

**流程 B：编辑闹钟**
1. 点击闹钟卡片 → 进入详情页
2. 可修改时间、参数、重试策略
3. 保存后立即生效

**流程 C：查看日志**
1. 点击闹钟卡片右上角「日志」图标
2. 进入日志页面，按时间倒序展示每次执行记录
3. 支持按成功/失败筛选

### 4. 功能 UI 设计（基于 animal-island-ui）

#### **整体视觉风格**
- **主题**：温暖、可爱、圆润（动森岛屿风）
- **颜色**：主色使用柔和的青绿 + 米白 + 浅木色
- **圆角**：大量使用较大圆角（12px~20px）
- **图标**：使用可爱风格图标（闹钟、小岛、树木、动物元素点缀）
- **支持**：Dark / Light 模式 + 中英双语

#### **主要页面结构**

**1. 首页（闹钟列表页）** —— 最重要页面
- 顶部：应用标题 + 设置按钮（齿轮图标）
- 中间：闹钟卡片列表（网格布局）
  - 每个卡片显示：
    - 闹钟名称
    - 下次触发时间
    - 状态（运行中 / 已暂停）
    - 最近执行结果（成功/失败 + 时间）
  - 卡片右上角：开关（启用/暂停）
  - 卡片底部：快捷操作按钮（立即执行 / 日志 / 编辑）
- 右下角：大大的「+ 新建闹钟」浮动按钮（圆形，可爱风格）

**2. 新建 / 编辑闹钟页**
采用**单页表单** + **步骤感**设计（不做多步向导，保持简单）：

- **基本信息区**（闹钟名称 + 时间设置）
- **任务配置区**（最重要，用卡片包裹）
  - Binary 输入框（支持自动补全常用命令）
  - 参数输入框（多行文本）
  - 环境变量（可折叠的 Key-Value 列表）
- **高级设置区**（可折叠）
  - 重试间隔选择（下拉或 segmented control）
- 底部：保存按钮（大按钮，醒目但不突兀）

**3. 日志页面**
- 时间轴样式展示每次执行记录
- 每条记录包含：时间、状态（成功/失败）、耗时、重试次数
- 点击可展开查看完整输出和错误信息

**4. 设置页**（简洁）
- 主题切换（Light / Dark / 跟随系统）
- 语言切换
- 启动时最小化到托盘
- 日志保留天数

### 5. 交互与反馈设计要点

| 场景 | 交互方式 | 反馈 |
|------|----------|------|
| 新建成功 | - | 顶部出现可爱 toast（“闹钟已设置好啦～”） |
| 执行成功 | - | 卡片上短暂显示绿色成功标记 |
| 执行失败 | - | 卡片显示红色失败标记 + 重试倒计时 |
| 手动执行 | 点击后弹出可爱确认弹窗 | 执行中显示加载状态 |
| 暂停闹钟 | 开关切换 | 卡片变灰，显示“已暂停” |

---

### 总结：核心设计目标

- **让用户感觉像在设置手机闹钟一样简单**
- **把“执行命令”包装成“闹钟要干的事”**
- **界面可爱但不影响信息传达**
- **后台安静运行，不打扰用户**
